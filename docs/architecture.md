# Архитектура

Одностраничное приложение на Vite + React 18 (без роутера — навигация по якорям).
Всё рендерится клиентом; сервер не требуется, деплой — статикой.

## Дерево композиции

```
main.tsx
└─ <App>
   └─ <I18nProvider>            // локаль + активный словарь (context)
      ├─ <HudCursor/>           // fixed overlay, только точный указатель
      ├─ <StarField/>           // fixed canvas-фон (z-0)
      ├─ <Nav/>                 // fixed, телеметрия + язык + меню
      ├─ <main z-10>
      │  ├─ <ScrollyHero/> | <StaticHero/>   // выбор по reduced-motion
      │  ├─ <Missions/>
      │  ├─ <Fleet/>
      │  ├─ <Route/>
      │  ├─ <Crew/>
      │  └─ <Contact/>
      └─ <Footer z-10>
```

## Слои и ответственность

| Слой | Директория | Ответственность |
| --- | --- | --- |
| Данные/контент | `src/i18n/` | Все тексты, типобезопасный словарь, выбор локали |
| Утилиты | `src/lib/` | Чистая математика (`easing`), константы, React-хуки |
| Примитивы | `src/components/primitives/` | Переиспользуемые эффекты и обёртки движения |
| Секции | `src/components/sections/` | Контентные блоки лендинга |
| Оболочка | `src/components/` | Курсор, фон, навигация, хиро, футер |

Правило зависимостей: секции и оболочка зависят от примитивов, хуков и i18n; примитивы —
только от хуков и easing; `lib/` ни от чего внутри `components/`. Односторонний поток.

## Ключевые потоки данных

**Локаль.** `I18nProvider` хранит `locale`, отдаёт мемоизированный `t` (активный
словарь). Компоненты читают через `useI18n()`. Смена языка пишет в `localStorage` и
обновляет `<html lang>` / `<title>` / `meta[description]`.

**Прогресс скролла.** Три независимых механизма, все на rAF-троттлинге и с cleanup:

- `ScrollyHero` — собственный rAF: маппит прогресс закреплённой 600vh-секции в
  `video.currentTime` с инерцией (lerp).
- `usePinProgress` — прогресс сквозь закреплённую секцию (маршрут-таймлайн).
- `useViewportProgress` — прохождение элемента через вьюпорт (параллакс фонов).

**Появление.** `useInView` (IntersectionObserver, one-shot) → `Reveal` и count-up флота.
Наблюдатель отключается после первого пересечения.

**Активная секция.** `useActiveSection` (IntersectionObserver по id) подсвечивает
навигацию и питает индикатор фазы в телеметрии.

## Производительность

- Один rAF-цикл на подсистему; StarField ставится на паузу при `visibilitychange`.
- DPR canvas ограничен (`STARFIELD.maxDpr`), плотность звёзд зависит от вьюпорта.
- Курсор и позиции пишутся напрямую в DOM (`transform`), минуя ре-рендеры React.
- lucide импортируется по иконкам (tree-shaking). Бандл ≈72 КБ JS gzip.

## Совместимость / деплой

Относительный `base: './'` + хелпер `asset()` (`BASE_URL`) → сборка работает в корне и в
подпапке GitHub Pages без изменений. Автодеплой — `.github/workflows/deploy.yml`.
