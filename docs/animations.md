# Спецификация движения

## Единый язык (v4) — читать первым

Три жеста покрывают весь сайт. Новых изобретать не нужно:

| Жест | Примитив | Для чего |
| --- | --- | --- |
| Строки из-под линий | `LineReveal` | ВЕСЬ текст и блоки |
| Фото-проявка | `MediaDevelop` | ВСЕ изображения |
| Меловой штамп сцены | `GhostScene` | Номер сцены в каждой секции |

- **Монтажная склейка:** уходящая секция садится (scale 0.985) и темнеет в
  последние 18% прохода — `SectionShell` делает это сам, off на low-tier.
- **Световая режиссура:** у каждой секции свой ключевой свет (`light="left|top|right|bottom"`),
  чтобы страница не была плоской.
- **Живые бары:** `FILM.barWide` на hero/финале, `FILM.barSlim` в контенте,
  `barHidden` при скролле вниз на таче (`useChromeVisible`).

Язык движения: медленно и весомо, кинематографичные склейки, warp-переходы между
секциями. Всё deterministically деградирует под `prefers-reduced-motion`.

## Кино-хром (CinemaChrome + FilmLeader)

- **FilmLeader** — каунтдаун 3-2-1 в стиле ракорда плёнки при первом входе:
  кольца, вращающийся радиус (spin за `FILM.leaderStepMs`), pop-in цифры
  (`leader-num`). Раз за сессию (`sessionStorage`), скип любым вводом
  (клик/клавиша/wheel/touch), не рендерится при reduced-motion.
- **Letterbox-бары** — фиксированные верх (h-12) и низ (h-10). Нижний живёт:
  сцена `SC.0X` + подпись меняются по активной секции (анимация `caption-in`
  через re-key), SMPTE-таймкод `00:MM:SS:FF` маппится из scrollY на плёнку
  `FILM.reelSeconds`×`FILM.fps` и пишется в DOM напрямую из rAF (без ре-рендеров).
- **Полноэкранное меню «затвор»** — две void-панели съезжаются сверху/снизу как
  обтюратор (700ms), затем слейты сцен `SC.02–06` wipe'ом раскрываются через
  clip-path (300ms + 90ms/пункт), пунктирные лидеры до подписей, гигантский
  призрачный номер текущей сцены фоном. Scramble на hover/focus, Esc/клик
  закрывают, body scroll-lock.
- **PlanetRise (финал)** — реальный Марс: 2K-карта (Solar System Scope, CC BY 4.0)
  даунсемплится в offscreen-текстуру, оборачивается в сферу с терминатором и
  тонким пыльно-оранжевым ободом атмосферы; едва заметное вращение (цикл только
  когда секция видима), параллакс-подъём — CSS-transform. Reduced — стоп-кадр.
- **Стиллы флота** — реальные NASA-фото бортов, на лету переведённые в холодный
  дуотон (grayscale → mix-blend-color + soft-light циановый лифт), в тикованной
  рамке с mono-подписью; re-key на смену борта.
- **GhostScene** — гигантский номер сцены «штампуется» при входе (scale 1.12→1 +
  fade) и дальше дрейфует параллаксом за контентом; слои дрейфа и штампа
  разделены, чтобы скролл не лагал.
- **Титры секций** — заголовок выезжает из-под маски, а трекинг «садится» из
  0.34em в норму (переход letter-spacing) — классический кино-титровый жест.

## Хиро — три монтажные версии

Выбор делает `useHeroCut()` в `App.tsx`. Версии рассказывают одну историю, но
разными средствами — это не деградация по кнопке, а разный монтаж под носитель.

### `scrub` — десктоп (`ScrollyHero`, 720vh)

Закреплённая секция. Прогресс маппится в `video.currentTime` с инерцией (lerp
`MOTION.scrubLerp`). Три текстовых блока:

- **A** (0–0.01) — split-хиро: заголовок слева, tagline справа, дата снизу.
- **B** (0.40–0.65) — брифинг миссии: покадровый word-cascade (`Words`), затем плавный
  выход.
- **C** — финал: `FINALE_START` 0.9, разворот за `FINALE_SPAN` 0.05, затем выдержка
  и только на `CUT_START` 0.975 — склейка. Раньше окна разворота и склейки
  перекрывались, и финал не успевали прочитать.

Idle-луп (`intro_loop.mp4`) играет над первым кадром, гаснет на первом скролле, возвращается
только когда scrub-время осело в нуле — стык без рывка.

### `mobile` — телефоны и тач (`MobileHero`, 320vh)

**Ни одного `<video>` в DOM.** Скраб — это запись `currentTime` каждый кадр;
телефон не успевает seek'ать внутри 26-мегабайтного файла, декодер встаёт, и
7 экранов закреплённого скролла выглядят как зависшая страница. Поэтому на
телефоне идёт последовательность постеров (`poster_start` → `poster_end`,
~250 КБ вместе): ближний кадр наезжает `scale 1 → 1.16`, дальний проявляется
на `p` 0.52–0.7, поверх — те же биты, что и на десктопе (`BEATS`).

Кадр стартует в тикованной кино-рамке (`FRAME_SCALE_FROM` 0.85) и раскрывается
до полного к `p` 0.22. Общий словарь движения обоих хиро — `hero/heroMotion.tsx`
(`rev`, `revB`, `MaskLine`, `Stat`, `FrameTick`), чтобы тайминги совпадали.

### `static` — reduced-motion (`StaticHero`)

Постер + весь текст сразу, без пиннинга, видео и склеек.

## Переходы между секциями

- **StarField «кино-оптика»** — глобальная соединительная ткань: три слоя глубины
  (дальние точки → звёзды в фокусе с дифракционными крестами → расфокусные
  боке-диски с «дыханием фокуса»), дымка туманности, никогда не повторяющийся
  Lissajous-дрейф «камеры», параллакс от скролла и курсора (только точный
  указатель), depth-scaled motion blur по скорости скролла, редкий метеор
  (первый через 5–9 с, дальше раз в 15–25 с), спутник с мигающим маячком,
  диагональная полоса-туманность, далёкая спиральная галактика на самом
  глубоком слое и «гипер-рывок» при резком скролле (усиленные полосы +
  аддитивное циановое свечение от центра). Все спрайты пре-рендерятся в
  offscreen-canvas — в кадре только drawImage/arc.
- **SectionShell atmosphere** — атмосферный градиент секции медленно дрейфует
  (параллакс) и «дышит» ярче в центре вьюпорта.
- **Reveal** — вход контента одним из вариантов: `rise` (подъём+блюр), `focus`
  (масштаб+расфокус→фокус — кино-склейка), `clip` (шторка clip-path), `fade`.
- **Contact** — театральный «восход планеты»: дуга лимба с холодным ледяным рим-светом
  параллаксом поднимается снизу.

## Микро-взаимодействия (hover)

| Эффект | Примитив | Где |
| --- | --- | --- |
| Bracket-кнопка (уголки разъезжаются) | `CineButton` | все CTA |
| Магнитное притяжение | `MagneticButton` (внутри CineButton) | CTA |
| 3D-наклон + скользящий блик | `TiltCard` | приборная схема флота |
| Scramble-декодирование | `ScrambleText` | пункты полноэкранного меню |
| Аккордеон-досье (`grid-template-rows` 0fr→1fr) | CSS grid | леджер миссий |
| Луч-подчёркивание табов | CSS (`scale-x`) | табы флота |
| Count-up характеристик | `useCountUp` | флот |
| Имя → accent на hover строки | CSS (`group-hover`) | титры экипажа |

Все — только для точных указателей (`usePointerFine`) и вне reduced-motion; тач получает
статичные, полностью доступные варианты.

## Курсор (HudCursor)

Визир-прицел: быстрая точка + инерционное кольцо (lerp `MOTION.cursorLerp`), перекрестие,
угловые тики при «захвате» интерактивных элементов (`[data-cursor]`), живой readout
координат. Позиция и текст пишутся прямо в DOM — без ре-рендеров. Off на тач/reduced-motion;
нативный курсор скрывается только на точных указателях (`html.has-custom-cursor`).

Разметка для захвата: добавить `data-cursor="..."` и опционально `data-cursor-label="..."`
на интерактивный элемент.

## Правила производительности и доступности

- Один rAF на подсистему, cleanup обязателен; canvas на паузу вне вкладки.
- Анимировать только `transform`/`opacity`/`filter`/`clip-path`; `will-change` — точечно.
- IntersectionObserver-появления — one-shot (отключаются после первого хита).
- `prefers-reduced-motion`: canvas и курсор off, появления мгновенные, хиро статичный,
  таймлайн маршрута — вертикальный список без пиннинга.

## Тюнинг

Числовые параметры движения собраны в `MOTION` и `STARFIELD` (`src/lib/constants.ts`) —
менять оттуда, не по месту.
