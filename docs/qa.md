# QA — процесс качества

Как проект проверяется: автоматические ворота, ручные чек-листы и журнал того,
что уже верифицировано. Цель — ни один коммит в `main` не ломает типы, сборку,
адаптив или доступность.

## Автоматические проверки (CI)

| Ворота | Где | Когда |
| --- | --- | --- |
| Строгая проверка типов (`tsc --noEmit`, без `any`) | `ci.yml` / `deploy.yml` | Каждый push и PR |
| Прод-сборка (`npm run build`) | `ci.yml` / `deploy.yml` | Каждый push и PR |
| Деплой на GitHub Pages | `deploy.yml` | Push в `main` |

Локально перед коммитом:

```bash
npx tsc --noEmit   # типы
npm run build      # сборка (включает tsc)
npm run preview    # смоук собранной версии
```

## Ручной чек-лист перед релизом (Definition of Done)

- [ ] **Адаптив:** 375px и 1280px, горизонтального переполнения нет
      (`document.documentElement.scrollWidth === clientWidth`).
- [ ] **Reduced motion:** в DevTools → Rendering → `prefers-reduced-motion: reduce` —
      статичный хиро, вертикальный таймлайн, без canvas-анимаций и курсора.
- [ ] **Клавиатура:** Tab обходит все интерактивы с видимым focus-ring; табы флота
      слушают стрелки/Home/End; Esc закрывает мобильное меню.
- [ ] **i18n:** переключение RU ⇄ UZ меняет весь контент, `<html lang>`, `<title>`;
      выбор переживает перезагрузку (localStorage).
- [ ] **Консоль:** ноль ошибок на полном прокруте страницы.
- [ ] **Ссылки:** все якоря секций и внешние CTA (Telegram, email, GitHub) ведут куда надо.
- [ ] **Медиа:** hero-видео скрабится плавно, idle-луп возвращается на самом верху.
- [ ] **Токены:** новые цвета — только через токены (`grep -ri '#[0-9a-f]\{6\}' src/`
      не должен находить хардкода вне зеркал `mirrors --accent`).

## Журнал верификации (последний прогон — v4 «Awwwards-проход»)

- Типографика: один класс на заголовки, `clamp()` даёт 74.9px на 1280 и 33.3px
  на 375 — контраст масштаба без переносов; переполнение 0 на обоих вьюпортах.
- Тач-эргономика: аудит выявил 12 целей ниже 40px → добавлен `.tap-target`
  (невидимая зона 44px только при `pointer: coarse`).
- Миссии: досье раскрыто, траектория рисуется, бортжурнал печатается,
  живой статус в марсианском `rgb(224,154,106)`.
- Флот: 8 призрачных засечек сравнения на барах, слейт-табы, прощёлк проектора.
- Финал: посадочный талон (4 поля), титры слиты в секцию.
- Звук: тумблер с `aria-label`, по умолчанию выкл.
- Бары: 60/52 в широкой сцене, консоль без ошибок.

**Квирк среды:** в скрытой панели браузера IntersectionObserver и CSS-переходы
заморожены, а таймеры дросселируются до ~1/сек — `computed opacity` врёт.
Проверять инлайн-стили и классы, а не computed-значения.

## Мобильный аудит (375 × 667 и 390 × 844)

- Горизонтального переполнения нет ни в одной секции: `scrollWidth − clientWidth = 0`
  на обеих ширинах, в обеих локалях. Отдельно проверено на UZ — узбекские строки
  длиннее русских и первыми ломают узкие макеты.
- `<video>` на телефоне — **ноль элементов**; вес первой загрузки ≈ 90 КБ JS (gzip)
  + два постера ≈ 250 КБ. `loadEventEnd` ≈ 310 мс локально.
- Тач-цели: 11 элементов с `.tap-target`, каждый ≥ 44 px по эффективной высоте,
  попарных пересечений хит-зон нет (иначе соседи воровали бы тап).
- Меню: `aria-expanded` переключается, `aria-controls` ведёт на `#menu-shutter`,
  body-скролл блокируется, Esc закрывает, фокус входит внутрь **только** когда
  меню открыто (см. ниже).
- Все шесть секций отдают контент; появления `LineReveal` разрешаются
  синхронно при монтировании, если блок уже во вьюпорте.

## Ограничение среды проверки

Панель предпросмотра в Claude Code рендерится не всегда: при `document.hidden`
браузер полностью останавливает `requestAnimationFrame` (0 кадров за 2 с) и не
доставляет колбэки `IntersectionObserver`. В таком состоянии любые проверки
появлений, canvas и скролл-прогресса дают ложные «ничего не появилось».
Признак проверяется одной строкой — и без него замерам верить нельзя:

```js
let n = 0; requestAnimationFrame(function t(){ n++; requestAnimationFrame(t) })
setTimeout(() => console.log(document.hidden, n), 1000)   // ожидаем false и ≳ 30
```

Надёжные в таком режиме проверки: геометрия (`getBoundingClientRect`), инлайновые
стили, атрибуты и ARIA, вес ресурсов, фокус. Появления проверяются через
синхронный путь `useInView` — HMR-ремонт компонента при нужной позиции скролла.

## Известные ограничения

- `scrub.mp4` ≈ 26 МБ грузится **только на десктопе**; первый скролл-скраб ждёт
  `canplaythrough` (есть экран загрузки). На телефон файл не приходит вовсе.
- Скраб-видео требует поддержки быстрого seek (все современные браузеры ок).
- `inert` для закрытого меню требует Chrome 102+ / Safari 15.5+ / Firefox 112+;
  в более старых браузерах остаётся `aria-hidden` без снятия из таб-порядка.
- Формы нет намеренно — сайт статический, заявки идут в Telegram/email.

## Как сообщать о багах

Issue в репозитории по шаблону: шаги воспроизведения → ожидаемое → фактическое →
вьюпорт/браузер → скриншот. Для визуальных багов скриншот обязателен.
